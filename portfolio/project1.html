"Blue Note"



The assignment

The assignment was to build a responsive website of a given style (Blue Note in my case), including information and images related to the style as well as showing the overall process of creating the website, like creating style tile, moodboard, color scheme, typography and so on. There was a requirement to use responsive principles of front end web design like media queries, grid design and flexible content to build our website.

How I applied what I learned in practice

For the design of my website I used complementary colors. The basic dark blue color I picked from the Blue Note Record's logo, and as a second blue color I used one of it's shades. I chose two shades of green and a kind of orange-red color. I actually picked these colours from some of the gramophone covers made by Reid Miles - designer and creator of the most famous and iconic covers of Blue Note Records. The chosen colors are not so saturated, because I wanted my website to be in this Blue Note retro style. Blue and green colors are cold and the orange is warm and stands on the opposite side in the colour wheel /complementary colors/. I think this combination is characterized by a harmonious colour contrasts and suits well with my given style.
In the design of my website can be seen mainly four of the Gestalt principles, namely Common Region, Proximity, Similarity and Symmetry. I used as well a lot of contrast to separate from each other the different parts with different content.

The basic principles that I used in the coding of HTML and CSS are use of a semantic code typical for HTML5, like "main", "section", "article" and so on. To arrange the HTML elements I used the grid of rows and columns in CSS. Grid-based layout system, makes it easier to design web pages without a need of using so much floats and positioning.

Other two techniques that I used are "media queries" in the CSS and "meta" tag implemented in the HTML files. Media queries are used to add breakpoints in the CSS and to set a rule how different parts of the website to behave on different in size screens. I have two "media queries" in my code which arrange the HTML elements in different way for screens bigger or smaller than 800px in width.

The "meta" tag is new for HTML5 and it gives the browser instructions on how to control the page's dimensions and scaling, so we can be able to see the whole website content correctly on different in size devices.
